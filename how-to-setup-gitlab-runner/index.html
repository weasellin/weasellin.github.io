How to Setup GitLab Runner on Local

05 July 2017

The purpose of this note is to setup a GitLab Runner on local to run GitLab CI jobs in the developing stage.

Dockerized GitLab Runner

(recommended approach)

First of all, by following this document,
I tried to run a dockerized gitlab-runner,
and mounted local repository directory with
-v <local_repo_dir>:<local_repo_dir> and specified the working directory
-w <local_repo_dir> for the purpose of auto-load the CI config .gitlab-ci.yml.

$ docker run \
    -d \
    --name gitlab-runner-container \
    --restart always \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -v <local_repo_dir>:<local_repo_dir> \
    -w <local_repo_dir> \
    gitlab/gitlab-runner:latest

To see gitlab-runner usage

$ docker exec -it gitlab-runner-container gitlab-runner

NAME:
   gitlab-runner - a GitLab Runner

USAGE:
   gitlab-runner [global options] command [command options] [arguments...]

VERSION:
   9.3.0 (3df822b)

...

To run CI jobs

$ docker exec -it gitlab-runner-container \
    gitlab-runner exec docker \
    --docker-privileged \
    [job_name]

If you use the image from the private docker registry, you could add the --DOCKER_AUTH_CONFIG with the json string from the ${HOME}/.docker/config.json.

$ docker exec -it gitlab-runner-container \
    gitlab-runner exec docker \
    --docker-privileged \
    --env DOCKER_AUTH_CONFIG='$(cat ~/.docker/config.json | tr -d '\040\011\012\015')' \
    [job_name]

[Updated Jul. 10th 2017] By mounted repository directory explicitly, I resolved the issue mentioned below.

I got an error

Running on runner--project-0-concurrent-0 via
a853d0cb3c01... Cloning repository... fatal: repository '/usr/local/repo' does not exist ERROR: Job failed: exit code 1 FATAL: exit code 1

With some solutions searching, I found this issue.
After failure on the trial of the --docker-privileged and --docker-volumes parameters, I decided to leave it alone and try to really install GitLab Runner.

Original GitLab Runner

Follow this document to install

$ curl -L https://packages.gitlab.com/install/repositories/runner/gitlab-ci-multi-runner/script.deb.sh | sudo bash
$ sudo apt-get install gitlab-ci-multi-runner
$ gitlab-runner 
NAME:
   gitlab-runner - a GitLab Runner

USAGE:
   gitlab-runner [global options] command [command options] [arguments...]

VERSION:
   9.3.0 (3df822b)

If you want to execute docker commands in the job script:

  • DOCKER_AUTH_CONFIG should be set with the content of ~/.docker/config.json, if the job requires registry privileged.
  • image: docker:latest should be specified in .gitlab-ci.yml.
  • the /var/run/docker.sock should be mounted by the gitlab-runner exec docker option --docker-volumes.
  • the other environment variables which will be used in job configurations.

for example:

$ gitlab-runner exec docker \
    --env DOCKER_AUTH_CONFIG='{"auths":{"registry.private":{"auth":"xxx"}}}' \
    --env CI_PROJECT_PATH_SLUG='project/path' \
    --docker-volumes /var/run/docker.sock:/var/run/docker.sock \
    staging

from now on, you can run the GitLab runner locally to test your project in each stages and jobs.
