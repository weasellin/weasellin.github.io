How to Setup My Ghost Pages

03 July 2017 on develop

By this setup, you can run a Ghost service on local host, and publish the content to GitHub Pages.

Before this Setup

Follow the post to setup required environment of GitHub client, Python virtual environment, and Docker.

How to Setup Ghost

From this docker image, it's very easy to run a Ghost service by the following command. And be sure to create a directory $GHOST_DATA_DIR whatever you prefer to keep data of the Ghost.

$ mkdir -p $GHOST_DATA_DIR
$ docker run \
      --name my-ghost \
      -p 2368:2368 \
      -d \
      -v $GHOST_DATA_DIR:/var/lib/ghost \
      ghost

Then, go to http://localhost:2368/ghost/ to manage the Ghost blog.

In addition, if your host can be accessed from a domain name or a public IP, by setting the config.development.config in the $GHOST_DATA_DIR/config.js, you can configure your blog to expose to http://<ghost_host>:2368.

$ docker restart my-ghost

to apply the changed configuration.

How to Publish to GitHub Pages

  • Follow this tutorial to create a <username>.github.io repository.

Then, use the Buster tools to setup, generate, and deploy the Ghost contents to the Github Pages.

# buster incompatible to python3
$ pyenv activate myenv-2.7.8
# install from my fork of buster
$ pip install git+https://github.com/weasellin/buster.git
$ cd $GHOST_DATA_DIR
$ buster setup --gh-repo=git@github.com:<username>/<username>.github.io.git
$ buster generate --domain '<ghost_host>:2368' --target-domain '<username>.github.io'
$ buster deploy
Good job! Deployed to Github Pages.

After deployment, the Pages in https://<username>.github.io should be updated.

Notice that, here is my fork of the buster to fix links with:

  • replacing <ghost_host>:2368 with <username>.github.io
  • using scheme relative URLs (removing http:, https: in the front of the //)

Update Publish From Ghost

$ pyenv activate myenv-2.7.8
$ cd $GHOST_DATA_DIR
$ buster generate --domain '<ghost_host>:2368' --target-domain '<username>.github.io'
$ buster deploy
Good job! Deployed to Github Pages.
